Command Line Parameters

Advanced users can execute specific TRIOS commands via the command prompt without having to open TRIOS software.

Command Line Parameters

/Geometry <geometry name>

Specifies the geometry to load – this will be the name of the geometry as displayed in the TRIOS geometry manager.

 

/Procedure <procedure name>

Specifies the name of the TRIOS procedure file to load (.tprc). If the path is not specified, the default TRIOS procedure directory for the instrument type will be used. A .tprc extension will be added if not defined in the command line.

 

/Sample <sample name>

The sample name.

 

/ResultsFile <results file name>

The path and file name for the results. This is optional if the file name definition in TRIOS includes the sample name.

 

/Export <export type>

The type of export required. The file will be stored in the same location as the results file with the appropriate file extension.

 

/Run <true/false>

Whether to automatically start a run after TRIOS starts.

 

/Close <true/false>

Whether to close TRIOS after the run.

 

/ErrorFile <file name>

The path and file name for the error file. This will be a text file which will contain any error information. If the run completes successfully this will contain “OK” in the first line.

 

/Analysis <model1#model2#...#modelN>

The specified analysis model(s) will be run on each step in the document generated in the run. An xml file will be generated with the same name as the results file generated but with “ –analysis” appended to the name and with an “xml” extension. Supported models are HerschelBulkley and PowerLawFlow at present.

 

/IP <ipaddress>

The IP address of the instrument

 

/Type <instrumentType>

The instrument type (e.g. ARG2, DHR3)

 

/<instrument serial number>

Specifies the instrument serial number – if this is used /IP and /type are not required.

 

/HideConnForm <true/false>

Whether to hide the connection form when starting up (for automation this should be set to true)

 

/LoadFileExportToTextAndExit <filename1>#<filename2>#<filename3> ….etc.

The specified file(s) are loaded, exported to text (all steps, default spreadsheet variables) and TRIOS is closed.

 

/ProcedureSequence <Discovery sequence file>

The specified sequence file is loaded into the running queue and started. When connected to a non-Discovery instrument the command is ignored.
